  - [ ] 저속 감지 테스트 (<4 km/h → 시간 모드)
  - [ ] 고속 감지 테스트 (≥4 km/h → 거리 모드)

### 3. **요금 계산 검증**
- [ ] **기본 요금**: 첫 미터는 3,000원에서 시작 ✓
- [ ] **초기 단계**: 첫 1,600m = +100원 → 계량화 모드로 전환
- [ ] **거리 모드**: 131m마다 +100원 (속도 ≥4 km/h)
- [ ] **시간 모드**: 30초마다 +100원 (속도 <4 km/h)
- [ ] **요금 표시**: 애니메이션이 부드럽고 주행 거리계가 올바르게 증가

### 4. **성능 & 안정성**
- [ ] **메모리 사용**: 장시간 GPS 세션 중 누수 모니터링
- [ ] **배터리**: GPS가 배터리를 과도하게 소모하지 않는지 확인
- [ ] **충돌 테스트**: 권한 거부, GPS 손실 시 충돌 없음
- [ ] **타입 안전성**: 빌드 전에 TypeScript 체크 실행

## ⚠️ 알려진 문제 / 보완할 부분

### 우선순위: 높음
1. **개인정보 보호정책 없음**
   - **조치**: 공개 개인정보 보호정책 URL 생성
   - **내용**: GPS 데이터 사용 및 보관 방식 설명 필수
   - **기한**: 제출 전 반드시 완료

2. **지원/랜딩 페이지 없음**
   - **조치**: 간단한 웹사이트 또는 GitHub Pages 생성
   - **URL 예시**: `https://yourdomain.com/taxi-meter`
   - **내용**: 앱 설명, 지원 이메일, 개인정보 보호정책 링크

3. **한글 지원**
   - **현재**: GPS 권한 메시지는 한글 ✓
   - **부족한 부분**: 
     - 한국 스토어용 한글 앱 이름, 설명
     - 상태 패널 텍스트 (이중 언어 지원 고려)
   - **조치**: 한국 앱스토어에 먼저 제출할지 결정

### 우선순위: 중간
1. **권한 거부 처리**
   - **현재**: GPS 권한 거부 시 오류 배너 표시
   - **확인**: 배너 디자인이 레트로 미학과 일치하는지
   - **고려**: "다시 시도" 또는 "설정 열기" 버튼 추가

2. **엣지 케이스**
   - **테스트**: 주행 중 GPS 신호 손실 시 어떻게 되나?
   - **테스트**: 휴대폰 전원 손실/앱 충돌 시 어떻게 되나?
   - **테스트**: 매우 긴 주행 (24시간 이상 세션)

3. **접근성**
   - **부족**: 명시적인 접근성 기능 없음
   - **고려**: 색맹 (속도 수준이 색상 사용?)
   - **고려**: 모든 사용자가 읽을 수 있는 글꼴 크기

### 우선순위: 낮음
1. **분석 & 충돌 보고** (선택사항)
   - Sentry 또는 유사 서비스로 오류 추적 고려
   - 선택사항: 기본 분석을 위해 Firebase 사용

2. **앱 검토 참고**
   - **iOS용**: GPS 사용이 필수 기능임을 언급
   - **Android용**: 표준 지침이 자동 승인해야 함
