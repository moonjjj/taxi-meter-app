## 1. 프로젝트 개요

- **프로젝트 이름**: 택시 미터기 스타일 러닝 트래커 (가칭)
- **핵심 컨셉**:  
  - 2000년대 초반 택시 미터기 UI를 모티브로 한 인터페이스  
  - 사용자의 **실제 이동 속도**(러닝/걷기/자전거 등)를 기반으로 요금이 올라가는 형태  
  - 화면 중앙 혹은 상단에 **달리는 말(러닝 마스코트) 애니메이션**을 배치하여, 속도에 따라 움직임이 달라지도록 표현  

- **목표 플랫폼(가정)**:  
  - 1차: 웹(모바일 브라우저 우선)  
  - 2차(선택): 모바일 앱(React Native 등)으로 확장 가능하도록 UI/로직 분리

---

## 2. 핵심 요구사항 요약

- **2000년대 초반 택시 미터기 스타일 UI**
  - 실제 택시 미터기의 레이아웃과 색감(검정/빨강 LED, 세그먼트 숫자 등)을 참고
  - 다음과 같은 정보가 항상 노출
    - 기본요금
    - 현재 누적 요금
    - 경과 시간
    - 현재 속도(옵션)

- **실제 이동 속도 기반 동작**
  - GPS 등을 이용해 사용자의 이동 속도를 주기적으로 측정
  - 측정된 속도에 따라 요금 증가 속도 조절

- **달리는 말 애니메이션**
  - 화면 상단 또는 중앙에 배치
  - 속도에 따라 애니이션 재생 속도 또는 프레임 변경
  - 정지/저속/중속/고속의 3~4단계 정도로 상태를 나눔

- **미터기 기능**
  - 시작/정지/초기화 기능
  - 실제 택시 미터기처럼, **기본요금 + 시간/속도 기반 요금**을 계산

## 3. UI 설계 (1차 버전)

### 3.1 레이아웃 개략

- **상단 영역**
  - 앱 제목 또는 간단한 로고
  - 설정/정보 버튼(아이콘)

- **중앙 영역**
  - 달리는 말 애니메이션(메인 비주얼)
  - 속도에 따라 애니메이션 변화

- **하단 미터기 영역**
  - 좌측: `기본요금` 고정 표시
  - 중앙: `현재 요금`(가장 강조된 숫자, 세그먼트 스타일)
  - 우측 상단: `경과 시간`(MM:SS 혹은 HH:MM:SS)
  - 우측 하단: `현재 속도 표시`(km/h 또는 m/s, 옵션)

- **컨트롤 영역**
  - 큰 버튼 1~3개
    - START / PAUSE / RESET (상태에 따라 라벨 변경)

## 4. 기능 요구사항 (Functional Requirements)

### 4.1 기본 미터기 동작

- **F-1: 측정 세션 시작**
  - START 버튼을 누르면:
    - 타이머 시작 (경과 시간 0부터 증가)
    - GPS/센서 기반 속도 측정 시작
    - 기본요금을 `현재 요금`에 즉시 반영
    - 달리는 말 애니메이션이 "달리기" 상태로 전환

- **F-2: 측정 일시정지/재개**
  - PAUSE 버튼:
    - 타이머 일시정지
    - 속도 측정 중단 또는 값 고정
    - 애니메이션을 "정지" 또는 "대기" 상태로 전환
  - RESUME(또는 START) 버튼:
    - 다시 타이머와 속도 측정 재개

- **F-3: 측정 종료 및 초기화**
  - RESET 버튼:
    - 타이머 0으로 초기화
    - 현재 요금 초기화 (기본요금만 남길지, 완전 0으로 할지 정책 필요)
    - 이동 거리/속도 기록 초기화
    - 애니메이션을 "대기" 상태로 전환

### 4.2 요금 계산 로직

- **F-4: 기본요금 표시**
  - 설정값(예: 3,000원)을 `기본요금` 영역에 항상 표시
  - START 시점에 `현재 요금`은 `기본요금`으로 세팅

- **F-5: 시간 기반 요금 증가**
  - 기준 시간 단위(예: 30초/1분)를 설정
  - 기준 시간마다 일정 금액(예: 100원) 증가
  - 사용자의 **속도에 따라** 증가 빈도 또는 증가 금액을 가중치로 조정
    - 예시 1: 빠르게 달릴수록 `요금 증가 주기`를 짧게 (ex. 1분 → 30초)
    - 예시 2: 속도 구간에 따라 `추가 가중치` 부여  

- **F-6: 속도 기반 가중치 설계(초안)**
  - 속도 구간 예시:
    - 0 ~ 4 km/h: 걷기(저속)
    - 4 ~ 10 km/h: 조깅(중속)
    - 10km/h 이상: 러닝(고속)
  - 각 구간마다 시간당 요금 증가량 계수를 다르게 적용
    - 저속: 기본 요금 증가 속도
    - 중속: 1.5배
    - 고속: 2배
  - 1차 버전에서는 단순한 계수 기반으로 구현 후, 추후 튜닝

### 4.3 애니메이션 연동

- **F-7: 속도에 따른 애니메이션 상태 제어**
  - 속도 구간별로 다른 애니메이션 사용 또는 재생 속도 변경
    - 저속: 걷는 말
    - 중속: 달리는 말(보통 속도)
    - 고속: 매우 빠르게 달리는 말(또는 이펙트 추가)

- **F-8: 정지 상태 처리**
  - 속도가 0에 가까운 상태(예: 0~0.5 km/h)는 "정지" 또는 "대기" 애니메이션
  - 일정 시간 이상 정지 시, UI에 "대기 중" 텍스트 등을 표시(선택)

## 10. 요금 계산 로직 & 숫자 롤링 표시 (2000년대 미터기 스타일)

> **원칙**: 요금 계산 로직과 숫자 표시 애니메이션 로직을 **분리**하여 구현한다.

### 10.1 목표

- **실제 요금 계산**은 기존 로직대로 정확하게 처리
- **화면에 보이는 요금**은 예전 미터기처럼 "숫자가 떨어지는(롤링)" 애니메이션 후 **100원씩** 증가
- 100원 증가할 때마다 미터기 숫자가 한 번 롤링되는 느낌
- 한 번에 여러 번 증가가 발생하면(예: +300원) **100원씩 N번 순차 애니메이션** 처리

### 10.2 핵심 상태값

| 상태값 | 타입 | 설명 |
|--------|------|------|
| `actualFare` | `number` | 실제 계산된 요금 (즉시 반영) |
| `displayFare` | `number` | 화면 표시 요금 (애니메이션 완료 후 반영) |
| `pendingFareSteps` | `number` | 아직 화면에 반영되지 않은 100원 단위 증가 횟수 (선택) |
| `isFareAnimating` | `boolean` | 현재 요금 숫자 롤링 애니메이션 진행 여부 |
| `fareStepQueue` | `number[]` 또는 queue | 각 +100원 단위 애니메이션 큐 |
| `lastFareIncrementAt` | `number \| null` | 마지막 증가 시점 (선택) |

### 10.3 동작 규칙

1. **요금 계산 엔진에서 +100원 증가 발생 시**
   - `actualFare += 100`
   - `displayFare`는 즉시 변경하지 않음
   - fare animation queue에 1 step 추가

2. **애니메이션 처리**
   - 큐가 비어있지 않고 `isFareAnimating === false`이면:
     - 큐에서 1 step 꺼냄
     - 숫자 롤링 애니메이션 시작 (예: 120~220ms)
     - 애니메이션 완료 후 `displayFare += 100`
     - 다음 큐가 있으면 이어서 반복

3. **한 번에 +N×100원 증가 시**
   - 큐에 N개 step을 넣고 순차적으로 처리
   - `displayFare`는 100원씩 단계적으로 올라가야 함

4. **동기화 보장**
   - 앱 성능 이슈로 프레임 드랍이 있어도 **최종적으로 `actualFare`와 `displayFare`가 반드시 일치**해야 함

### 10.4 애니메이션 요구사항

- 각 자릿수(만원/천원/백원/십원/일원) 중 **실제로 변하는 자릿수만** 롤링
- 최소 구현 버전에서는 전체 숫자 롤링처럼 보여도 무방
- 100원 단위 증가 시 **백원 자리 중심**으로 떨어지는 느낌
- CSS `transform: translateY` 또는 React Native `Animated`/`Reanimated`로 구현 가능하도록 구조화
- UI에서 "롤링 중" 상태를 알 수 있도록 콜백/상태 제공

### 10.5 구현 요청 사항

- **언어**: TypeScript
- **순수 로직 계층**: `FareDisplayAnimator` (클래스 또는 훅)로 분리
- **메서드 예시**:
  - `enqueueFareIncrement(steps: number)` — 100원 단위 step N개 큐에 추가
  - `processNextStep()` — 다음 step 처리 (애니메이션 트리거)
  - `onAnimationComplete()` — 애니메이션 완료 시 `displayFare` 갱신 및 다음 step 처리
  - `syncDisplayFareToActual()` — 복구용: `displayFare`를 `actualFare`로 강제 동기화
- 실제 요금 계산 엔진(`useTaxiMeterCalculator`)과 연결 예시 코드 작성
- React / React Native 둘 다 응용 가능하도록 **UI 의존성 낮게** 작성

### 10.6 예외 처리

- **stop/reset 시**: 큐 비우기 옵션 제공
- **앱 백그라운드/포그라운드 전환 후**: `displayFare`가 `actualFare`보다 뒤처졌으면 안전하게 동기화
- **연속 과금이 매우 빠르게 들어와도**: 큐가 꼬이지 않도록 처리
- **중복 애니메이션 방지**: `isFareAnimating` lock으로 한 번에 하나의 롤링만 실행

### 10.7 2000년대 미터기 느낌 — 숫자 롤링 UI 데이터 구조 제안

| 필드 | 타입 | 설명 |
|------|------|------|
| `digitColumns` | `{ place, currentDigit, nextDigit, offsetProgress }[]` | 자릿수별 컬럼 (만~일) |
| `place` | `number` | 자리 (예: 0=일, 1=십, 2=백, 3=천, 4=만) |
| `currentDigit` | `number` | 현재 표시 중인 숫자 (0~9) |
| `nextDigit` | `number` | 롤링 후 표시할 숫자 |
| `offsetProgress` | `number` | 0~1, 롤링 진행률 (translateY 또는 opacity 연동) |

- **최소 구현**: `displayFare` 한 개 값만으로 "전체 숫자 롤링"처럼 보이게 해도 가능
- **고급 구현**: 위 구조로 자릿수별로 변하는 자리만 롤링

### 10.8 샘플 시뮬레이션 시나리오

- **+100원** → 큐 1 step → `displayFare` 1회 롤링 후 +100
- **+300원** → 큐 3 step → `displayFare` 3회 순차 롤링 (100원씩)
- **정체 구간 후 연속 증가** → 큐에 쌓인 step이 순차 처리되며 `displayFare`가 `actualFare`에 수렴
- 로그 출력: `actualFare`, `displayFare`, `queue.length`, `isFareAnimating` 등으로 동작 검증
